01 · 현황 진단

관별로 매쓰플랫을 쓰는 방식이 다릅니다

같은 도구를 쓰지만 데이터가 쌓이는 방식이 제각각입니다. 특히 "학생 등록 없이 문제은행용"으로 쓰는 관은 학생 단위 데이터가 거의 남지 않아 생기부의 원천이 만들어지지 않습니다.

초등관

매쓰플랫 Basic(학생 추가) 또는 Premium
  • 학생을 등록해 출제·채점 → 학생별 데이터가 축적되는 형태
  • 유형분석·리포트 등 학습 데이터 활용 기반 존재
한계 · Basic은 30명 초과 시 인원당 과금, 데이터가 매쓰플랫 안에 갇혀 ERP·생기부로 흐르지 않음

중등 특목관

매쓰플랫 — 학생 등록 없이 문제은행용
  • 출제·인쇄 중심으로만 사용
  • 학생을 등록하지 않고, 따로 만든 자체 형식으로 관리 중
한계 · 자체 형식이라 데이터가 표준화·디지털화되지 않아 생기부로 연결하기 어려움

자사관

매쓰플랫 출제 → 오르조 업로드·출제 + 자체 개발 플랫폼(학부모 피드백)
  • 매쓰플랫으로 출제한 문제를 선생님이 오르조에 다시 업로드해 학생에게 출제
  • 오르조에 데이터는 쌓이나 학생 개별 성장 추이·오답 출제·취약유형 관리는 거의 안 됨
  • 리포트 기능 거의 없음 · 학부모 피드백은 별도 자체 개발 플랫폼으로
한계 · 데이터가 여러 도구로 분산·이중 업로드되고 학생 단위 성장·취약 데이터가 미흡 → 생기부 원천으로 부족

공통 문제

  • 관마다 사용 방식이 달라 데이터 표준이 없습니다 — 통합 생기부를 만들 수 없습니다.
  • 데이터가 학생 단위로 구조화되지 않으면 ERP(위드아카) 연동·종합 생기부가 불가능합니다.
02 · 목표 운영 모델

핵심은 "출제 도구"에서 "학생 데이터 자산"으로의 이동입니다. 개별 학생의 학습이 매일 데이터로 쌓이고, 그 데이터가 출제와 관리의 근거가 되며, 최종적으로 ERP를 통해 종합 생기부로 산출되는 구조입니다.

03 · 전환 당위성

왜 매쓰홀릭으로 통일해야 하는가

데이터화 학생관리와 생기부라는 목표 기준으로 보면, 매쓰홀릭은 "학생 단위로 매일 데이터가 쌓이는" 설계가 기본에 가깝습니다. 아래는 실제 화면·공식 자료로 확인한 근거입니다.

DATA · 일자형

학생 × 요일 달력형 학습현황

학생 행을 펼치면 그날그날의 과제·완료비율·정답률이 일자별로 누적됩니다. 일일 단위 개별 데이터가 자연스럽게 쌓이는 구조 — 생기부의 1차 원천.

FEEDBACK · 입문등급

AI 풀이채점 (개별 피드백)

학생 풀이를 단계별로 분석해 틀린 단계·필수 개념을 짚어줍니다. 라이트 단계부터 사용(포인트 차감). 개별 피드백 데이터가 학생별로 남습니다.

GROWTH · 시계열

진단평가 회차 운영

입학·주간·월간·단원 평가를 회차로 운영 → 성장 추이가 시계열 데이터로 축적됩니다. 모바일 보고서(카톡·SMS)로 일·주·월 리포트 자동 전달.

SOURCE · 근거출제

교재채점 · 연산 · 시험대장

시중교재 채점(빠른정답·단원별 정답률), 연산대장, 학교 기출 AI 변형까지 — 출제와 채점 자체가 학생 데이터로 환원됩니다.

STRATEGY · 프리미엄

유형 이해도 · 취약유형 과제

프리미엄에서 단원별 유형 이해도(누적 종합)·취약유형 자동 과제·유형학습 강의가 더해져 데이터 기반 개별 전략이 완성됩니다.

ERP · 생기부

학생 단위 데이터 = 생기부의 원천

모든 활동이 학생 단위로 구조화되므로, ERP(위드아카) 연동 시 종합 생기부로 환원하기 유리합니다. ※ 실제 연동 방식은 확인 필요(아래 참조)

04 · 핵심 비교 (입문 등급)

매쓰플랫 베이직 vs 매쓰홀릭 라이트

데이터화 관리 관점의 실질 차이만 정리했습니다. "학생 스스로 하는 유형학습(self-study)"은 양쪽 다 프리미엄이라 입문 등급에선 무승부입니다.

항목매쓰플랫 베이직매쓰홀릭 라이트
학생별 데이터 축적문항→유형 누적 단, 학생 등록 시 / 교사 전용 뷰일자(달력)형 + 문항·난이도·단원학생 행 펼침 → 일자별 학습
일일 · 달력형 관리유형격자·리포트 중심달력은 주로 출결학생×요일 달력 + 일일 리포트카톡·SMS 발송
개별 피드백 (AI 풀이채점)✗ (프리미엄)○ 단계 분석포인트 차감
진단평가 출제 도구✗ (유형분석·리포트로 대체)○ 입학·주간·월간·단원
유형 종합 진단 (교사용)△ 누적 격자교사 전용 · 30명 한도△ 채점 후 유형분석누적 "유형 이해도"는 프리미엄
학생 유형학습 (self-study)✗ 프리미엄✗ 프리미엄 → 양쪽 동일
연산△ 단원 내 포함○ 연산대장 전용
내신 · 기출 시험지○ 내신관(학교 기출)○ 시험대장(AI 원본 변형)
개념 동영상(강의)○ 단원별 개념 영상 포함문제풀이 동영상은 프리미엄✗ (강의는 프리미엄)2만 개 개념·유형해설 강의

ERP·생기부 연계 적합성출제도구 성격학생 미등록 시 데이터 공백학생 단위 데이터 구조※ 위드아카 연동 방식 확인 필요

실측 기준: 매쓰플랫(베이직 계정 직접 확인) · 매쓰홀릭(프리미엄 계정에서 라이트 해당 기능 확인) · 일부 등급 경계는 공식 안내(matholic.com) 기준.

05 · 관별 전환 전략

관별로 무엇을 어떻게 가져갈 것인가

관마다 현재 상태가 다르므로 전환 경로도 다릅니다. 공통 목표는 "학생 단위 데이터 표준화 → 생기부 연결".

초등관

  • 이미 학생 등록 기반 → 매쓰홀릭으로 데이터 표준 이전
  • 일일 달력형 관리 + 일일 리포트로 학부모 소통 강화
  • 규모(인원) 기준으로 라이트/프리미엄 결정

중등 특목관

  • 최우선 과제: 학생 등록 체계 도입
  • 문제은행 사용을 "채점·데이터 축적"까지 확장
  • 진단평가 회차로 특목 대비 성장 데이터 확보

자사관

  • 매쓰홀릭 + 생기부 흐름의 표준 케이스로 선(先) 적용
  • 유형 이해도·취약유형 과제(프리미엄)로 개별 전략 검증
  • 전관 확대 전 데이터→생기부 산출 검증
06 · 회의 진행

오늘 회의 안건

각 관의 현재 사용 방식을 듣고, 데이터화 비전을 공유한 뒤, 전환 당위성과 향후 진행을 함께 결정합니다.

PART 1

관별 현재 매쓰플랫 사용 방식 공유

초등관 · 중등 특목관 · 자사관 — 어떻게, 어디까지 활용 중인지, 학생 데이터가 실제로 쌓이고 있는지 점검

PART 2

데이터화 학생관리 비전 공유

개별 추적 → 근거 출제 → 데이터 관리 → 위드아카 연동 → 종합 생기부 목표 정렬

PART 3

매쓰홀릭 전환 당위성 검토

학생 단위 일일 데이터·AI 피드백·진단평가·생기부 연계 적합성 근거 검토

PART 4

향후 진행 로드맵 논의 · 결정

파일럿 관 선정, 라이트/프리미엄 결정, 위드아카 연동 확인, 전관 확대 일정

전환 로드맵(안)

STEP 1

현황·데이터 진단

관별 사용 방식·학생 데이터 축적 현황 파악

STEP 2

파일럿 적용

1개 관(자사관 권장) 매쓰홀릭 적용 · 등급 결정

STEP 3

ERP 연동 설계

매쓰홀릭 데이터 → 위드아카 연동 방식 확인·설계

STEP 4

전관 확대 + 생기부

표준 정착 후 확대 · 종합 생기부 템플릿 정의

회의 전 확인 필요 (정직한 미확정 사항)

  • 매쓰홀릭 ↔ 위드아카(과사람 ERP) 데이터 연동의 실제 지원 여부·방식 (API/내보내기 등) — 본 자료는 "데이터 구조상 유리"까지만 확인, 실연동은 양사 확인 필요
  • 관별 인원 기준 견적 — 매쓰홀릭 라이트 vs 프리미엄 (정찰가 비공개, 상담 필요)
  • 매쓰플랫 인원당 추가 과금 단가 — 현 비용 대비 전환 손익분기 산출용
  • 종합 생기부 산출 항목 정의 — 어떤 데이터를 어떤 형태로 학부모/내부에 제공할지
학생 데이터화 관리 전환 회의 자료 · 내부 검토용 · 수치/등급 경계는 실측 및 공식 안내 기준이며 확정 견적은 별도 협의 필요
